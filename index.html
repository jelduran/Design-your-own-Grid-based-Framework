Framework for Microverse

This is a CSS Framework created with Sass Preprocessor. It allows a grid system of maximum 12 columns with a base font size of 16 pixels and "Lucida Sans" as a default font family. Next are described available classes and how to use them!

Mesh

Classes mesh and mesh-boxed represents the main container of the framework. The only difference is that mesh-boxed has a max-width value of 1100px. For example all the elements in this webpage are contained in a mesh-boxed class element.

Boxes

Class box-n where n represents the number of columns that the box will occupy; if n is multiple of 12, the boxes will be able to fill the whole space, if not, the elements that doesn't fit will move behind the row if they overpass the available space. These classes can be set on any no-inline-type element. (Blue color is showing gutters in examples.)

Fill Boxes

Sometimes is neccessary to fill a tier or a box with a number of boxes of the same size. The class fillbox-n allow to do this with no need of assigning a class to every child element, n still represents the number of columns that any filler box will occupy; if n is multiple of 12, the boxes will be able to fill the whole space, if not, the elements that doesn't fit will move behind the row if they overpass the available space. The fillbox-n class only distribute elements of type div, header, section, article, aside and footer. (Blue color is showing gutters in examples.)

Breakpoints

This framework has breakpoints at 1024px, 720px, 576px and 480px minimum screen widths, represented by letters l, m, s and xs respectively. By default non-breakpoint classes works for a screen size smaller than 480px. Classes box-n and fillbox-n use breakpoints just by adding the corresponding letter at the end of the class name, for example, box-4s or fillbox-12l, for other classes that set CSS properties is neccessary to add -breakpoint either at the end or in the class name, for example, d-flex-m sets display:flex for an element at breakpoint m and p-t-s-1 sets padding-top: 0.25rem for an element at breakpoint s.

Margins

Margins are defined by class m-n where n is a number from 0 to 4 representing the size of the margin. This can be also defined for a given side using the notation m[-t | -l | -b | -r]-<0 | 1 | 2 | 3 | 4>, for example, m-t-2 sets margin-top in size 2. (Gray color is showing margins in examples.)

Paddings

Paddings are defined by class p-n where n is a number from 0 to 4 representing the size of the margin. This can be also defined for either a given side or a given breakpoint using the notation p[[-t | -l | -b | -r] | <-t | -l | -b | -r>[-l | -m | -s | -xs]]-<0 | 1 | 2 | 3 | 4>, for example, p-t-m-2 sets padding-top for breakpoint m in size 2. Colors

There are 7 framework colors for text, backgrounds and borders. Can be defined by the notation <white | gray | secondary | black | primary | accent | passed>[-bg | -border], for example, gray-bg sets gray element's background color, accent-border sets element's border color and only primary with no suffix, sets text color.
